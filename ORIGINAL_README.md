# 相识北洋 Python 版

这是“挑战3：相识北洋”的 Python 实现版本，使用 **Python 标准库**完成，不需要安装第三方依赖。

## 一、实现功能

- 个人自我介绍：昵称、简介、本机固定身份 ID。
- 校园网同学发现：同一 Wi-Fi / 校园网下通过 UDP 广播自动发现在线同学。
- 好友申请：向在线同学发送好友申请，对方可同意或拒绝。
- 好友管理：保存好友列表，展示在线 / 离线状态，支持好友分类和删除好友。
- 好友聊天：好友在线时使用 TCP 直连发送消息，支持编辑尚未送达的待发送消息。
- IP 改变后继续通信：好友身份使用固定 UUID，不依赖固定 IP；重新上线后自动更新 IP。
- 离线消息：好友不在线时消息进入待发送队列。
- 中转消息：如果双方不能直接在线遇到，程序会尝试通过共同在线好友中转消息；中间好友会暂存无法立即送达的中转消息，等待目标好友上线后继续投递。

## 二、运行环境

建议使用：

- Python 3.9 及以上
- Windows / macOS / Linux
- 两台设备连接同一个 Wi-Fi 或校园网

手机也可以尝试使用 Pydroid 3 等 Python 环境运行，然后在手机浏览器打开 `http://127.0.0.1:8765`。

## 三、运行方法

进入项目目录后执行：

```bash
python main.py
```

或 Windows 双击：

```text
run_windows.bat
```

运行后浏览器打开：

```text
http://127.0.0.1:8765
```

如果想让别人访问你这台设备的网页界面，控制台会显示类似：

```text
同设备局域网地址：http://192.168.1.23:8765
```

## 四、两台设备测试方法

1. 两台电脑连接同一个 Wi-Fi。
2. 两台电脑都运行：`python main.py`。
3. 两边浏览器都会打开“相识北洋”页面。
4. 修改昵称和自我介绍。
5. 等待几秒，左侧“在线同学”会出现对方。
6. A 给 B 发送好友申请。
7. B 在“好友申请”里点击同意。
8. 双方成为好友后即可聊天。
9. 关闭 B 的程序，A 继续给 B 发消息，消息会进入待发送队列。
10. 重新打开 B 的程序后，A 会自动尝试发送待发送消息。

## 五、常见问题

### 1. 看不到在线同学怎么办？

优先检查：

- 两台设备是否在同一个 Wi-Fi / 局域网。
- Windows 防火墙是否允许 Python 访问网络。
- 校园网是否禁止设备互访。部分校园网开启 AP 隔离，会导致局域网广播和 TCP 连接失败。

### 2. 端口被占用怎么办？

默认网页端口是 8765。可以改端口运行：

```bash
python main.py --http-port 8888
```

消息 TCP 端口默认会自动寻找 50124 附近可用端口，也可以手动指定：

```bash
python main.py --tcp-port 50130
```

### 3. 数据保存在哪里？

程序运行后会生成：

```text
beiyang_data/
├── profile.json
├── friends.json
├── messages.json
├── pending.json
├── relay_pending.json
└── requests.json
```

删除这个文件夹可以重置身份和聊天数据。
